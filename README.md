# AcaLite

AcaLite 是一个由一人公司主导、后期全面开源的轻量 AI 学术工具。

## 文档导航

- [系统架构设计（MVP）](docs/architecture.md)
- [下一步执行计划（从架构到实现）](docs/next-steps.md)

## 产品定位

- **可本地部署**：支持私有化运行，避免科研文献与草稿外流。
- **无数据泄露风险**：核心流程围绕本地处理设计，不依赖公有云保存用户数据。
- **聚焦核心痛点**：优先解决文献解读、本地检索、引用规范问题。
- **轻资产模式**：不建设重资产商业文献数据库，以开源引流 + 技术服务变现为主。

## 为什么做（适配一人公司）

1. **自身优势适配**
   - 人力有限，聚焦 Python/FastAPI/Vue 等技术开发。
   - 依托开源降低运营与获客成本。
2. **市场痛点适配**
   - 传统平台订阅成本高，且通常不支持本地部署。
   - 许多 AI 学术工具闭源，技术用户无法自定义。
3. **开源模式红利**
   - 通过 GitHub、V2EX 等社区快速传播与积累用户。
   - 形成“开源引流 -> 服务变现 -> 社区迭代”的闭环。
4. **风险可控**
   - 不做大规模文献爬虫与存储，规避版权与合规风险。

## 核心用户

### C 端核心用户

- 技术极客、独立研究者：有本地部署和可定制需求。
- 本硕博学生、青年教师：预算敏感，重视数据隐私和实用性。

### B 端种子用户

- 小型实验室、教研小组：需要低门槛部署与基础协同。
- 开源社区用户：可贡献反馈、插件与传播。

### 后期拓展用户

- 第三方开发者：围绕插件生态扩展能力。
- 中小科研机构：私有化部署 + 定制支持。

## MVP 功能范围（6 个月）

### 1) 本地文献管理与导入

- 支持本地 PDF 批量导入、分类归档（学科/时间/关键词）。
- 支持文献预览与基础文本标注。

### 2) AI 文献智能解读（核心卖点）

- 对接开源模型（Llama、Qwen）或第三方 API（OpenAI）。
- 自动生成摘要、核心观点、研究方法、结论。
- 提供中英文翻译与简易思维导图。

### 3) 智能检索与引用生成

- 基于 RAG 的本地检索（关键词 + 自然语言）。
- 支持 APA、MLA、GB/T 7714 等引用格式一键生成。

## 商业化路径

- **阶段一（冷启动）**：开源发布 MVP，获取真实反馈。
- **阶段二（增长）**：提供托管部署、技术支持、定制功能。
- **阶段三（生态）**：构建插件生态与社区协作机制。

## 非目标（当前阶段）

- 不做商业文献数据库与版权采购。
- 不做重运营导向的平台型服务。
- 不追求早期大而全功能。

## 里程碑建议

- **M1-M2**：完成本地导入、预览、索引基础能力。
- **M3-M4**：完成 AI 解读链路、翻译与摘要模板。
- **M5**：完成检索 + 引用格式生成。
- **M6**：打磨部署体验、发布开源版本与文档。

## 当前代码状态（已落地）

- `backend/`：FastAPI 后端骨架，包含以下接口：
  - `GET /api/v1/health`
  - `POST /api/v1/documents/import`
  - `POST /api/v1/retrieval/search`
  - `POST /api/v1/ai/summarize/{document_id}`
  - `POST /api/v1/citations/generate`
- `frontend/`：Vue3 前端骨架，包含文献导入页、检索页、设置页占位。
- `deploy/docker-compose.yml`：一键启动 `postgres + redis + api + web`。

## 快速启动

```bash
cd deploy
docker compose up --build
```

- 前端：`http://localhost:5173`
- 后端 OpenAPI：`http://localhost:8000/docs`
